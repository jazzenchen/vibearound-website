---
title: 远程编程
description: 从浏览器、手机、终端或消息频道继续本地 AI 编程 Agent 会话，而不把工作空间搬到云端 IDE。
---

VibeAround 里的远程编程，指的是远程接入本地工作空间。Agent、仓库、终端、凭据、包缓存和开发服务仍然保留在主机上；浏览器、手机、Web Terminal、消息频道和预览链接只是回到这台机器的受控入口。

## 适用场景

适合在这些情况下使用：

- 编程 Agent 已经在可信的笔记本、台式机、工作站或服务器上运行。
- 项目依赖本地凭据、本地服务、私有网络、硬件或桌面工具。
- 你离开键盘后仍需要审阅、批准或调整会话方向。
- 你希望获得远程可达性，但不想把仓库克隆到托管云工作空间。

## 推荐设置流程

1. 从 [下载](./download) 安装 VibeAround。
2. 按 [快速开始](./quick-start) 验证一个 Agent 可以本地运行。
3. 把项目加入工作空间。
4. 通过 [Agent 启动](./agent-launch) 开始会话。
5. 先启用一个远程入口：[会话接续](./session-handover)、[远程消息与 Web Terminal](./remote-messaging-terminal) 或 [实时预览](./live-preview)。
6. 启用隧道或公开链接前，先阅读 [本地优先安全](./local-first-security)。

## 远程入口

| 入口 | 适合用途 |
| --- | --- |
| Web Chat | 从浏览器继续短提示和会话控制。 |
| Web Terminal | 以接近 shell 的方式访问本地工作空间。 |
| 手机浏览器 | 离开电脑后的快速审阅、批准或调整。 |
| 消息频道 | 通过 Telegram、飞书/Lark、Discord、Slack、微信、钉钉、企业微信或 QQ Bot 异步查看和控制。 |
| 实时预览 | 审阅本地开发服务、Markdown、HTML 和生成产物。 |

## 安全检查

- 确认谁可以访问会话。
- 远程访问真正需要时再开启隧道。
- 将 Web Terminal 和消息机器人视为高权限控制面。
- 使用有范围限制的预览链接，而不是开放式访问。
- 不再需要输入的会话应停止或归档。
