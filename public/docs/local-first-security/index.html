Quick Start Handover Channels Providers Security

security

Local-first does not mean risk-free. Know the boundary.

VibeAround keeps agents and workspaces local, but remote access, IM channels, tunnels, and model providers still need careful setup.

What stays local

  • Your repositories and project files remain on your computer unless an agent or provider request includes their content.
  • Shell commands and local development servers run on your machine.
  • The local dashboard is served from your computer.

What can leave your machine

  • Prompts, code excerpts, files, or command output sent to a model provider by the agent you choose.
  • Messages exchanged through connected IM platforms.
  • Preview traffic when you expose a local service through a tunnel.

Access controls to care about

VibeAround uses local session authentication for daemon, WebSocket, and MCP-style access. Treat IM bot tokens, tunnel tokens, provider API keys, and generated handover codes as sensitive.

Safe operating habits

  1. Start with one workspace and one channel.
  2. Use dedicated bot credentials for VibeAround.
  3. Do not expose broad local services through tunnels unless you need them.
  4. Review agent output before applying changes to important projects.
  5. Rotate tokens when a device, channel, or tunnel is no longer trusted.
