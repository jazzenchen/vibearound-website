provider profiles

Switch agents, providers, and bridge routes without rewriting every CLI config.

Provider profiles help VibeAround act as a local AI switch for Claude Code, Codex CLI, Gemini CLI, Cursor CLI, Qwen Code, Kiro CLI, and OpenCode workflows.

What a profile controls

  • The agent or CLI that should handle the task.
  • The model provider or OpenAI-compatible endpoint to route through.
  • The workspace where the agent should run.
  • The bridge route used for Responses, Chat Completions, or Anthropic-style messages.

Common provider profile targets

VibeAround workflows commonly use first-party APIs, OpenAI-compatible endpoints, DeepSeek, Qwen/DashScope, Moonshot/Kimi, MiniMax, Z.AI/GLM, Google Gemini, OpenRouter, Azure OpenAI, and custom endpoints.

Switch during a conversation

Use /profile <name> to switch provider profile and /switch <agent> to switch enabled agent. This is the workflow to use when you are searching for Claude Code provider switching, CC Switch-style routing, or a local AI switchboard.
