handover

Move one local agent session between terminal, browser, and IM.

Session handover is the core remote-coding workflow: keep the agent running locally, then pick up the same conversation from another entry point.

Basic flow

  1. Start or continue an agent session in VibeAround.
  2. Send /handover from the active session.
  3. Copy the generated /pickup CODE command.
  4. Paste that command into a connected IM channel or browser session.
  5. Continue with the same local context.

Core commands

CommandUse it for
/helpShow available commands in the current channel.
/newStart a fresh conversation.
/switch <agent>Switch to another enabled agent, such as claude, codex, or gemini.
/profile <name>Switch provider or route profile.
/handoverExport the active session for pickup elsewhere.
/pickup <code>Resume a handed-over session.
/closeClose the current conversation.

Slack command prefix

Slack intercepts slash commands. Use /va or /vibearound as the command prefix, such as /va help or /va switch claude.
