本地优先不等于没有风险。先理解边界。

VibeAround 让 Agent 和 workspace 留在本地，但远程访问、IM 渠道、tunnel 和模型 Provider 仍然需要谨慎配置。

什么留在本地

  • 仓库和项目文件仍在你的电脑上，除非你选择的 Agent 或 Provider 请求中包含相关内容。
  • Shell 命令和本地开发服务在你的机器上运行。
  • 本地 Dashboard 从你的电脑提供访问。

什么可能离开你的机器

  • Agent 发送给模型 Provider 的 prompt、代码片段、文件或命令输出。
  • 通过 IM 平台交换的消息。
  • 通过 tunnel 暴露本地服务时产生的预览流量。

需要关注的访问控制

VibeAround 对 daemon、WebSocket 和 MCP 风格访问使用本地 session auth。请把 IM bot token、tunnel token、provider API key 和生成的 handover code 都当作敏感信息处理。

推荐安全习惯

  1. 先从一个 workspace 和一个渠道开始。
  2. 为 VibeAround 使用专用 bot credential。
  3. 除非必要，不要通过 tunnel 暴露过宽的本地服务。
  4. 在重要项目上应用 Agent 改动前，先 review 输出。
  5. 当设备、渠道或 tunnel 不再可信时，及时轮换 token。
