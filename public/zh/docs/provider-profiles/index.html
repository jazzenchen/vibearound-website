provider profiles

不用反复改 CLI 配置，也能切换 Agent、Provider 和 Bridge Route。

Provider Profile 让 VibeAround 成为 Claude Code、Codex CLI、Gemini CLI、Cursor CLI、Qwen Code、Kiro CLI 和 OpenCode 的本地 AI switch。

Profile 控制什么

  • 当前任务使用哪个 Agent 或 CLI。
  • 通过哪个模型 Provider 或 OpenAI-compatible endpoint 路由。
  • Agent 应该在哪个 workspace 运行。
  • 使用 Responses、Chat Completions 或 Anthropic-style messages 的哪条 bridge route。

常见 Provider Profile

常见工作流包括官方 API、OpenAI-compatible endpoint、DeepSeek、Qwen/DashScope、Moonshot/Kimi、MiniMax、Z.AI/GLM、Google Gemini、OpenRouter、Azure OpenAI 和自定义 endpoint。

在对话中切换

使用 /profile <name> 切换 provider profile，使用 /switch <agent> 切换已启用 Agent。如果你在搜索 Claude Code 切换、CC Switch、Codex CLI provider switch 或本地 AI 切换，这就是对应的工作流。
